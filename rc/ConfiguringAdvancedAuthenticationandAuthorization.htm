Exercise 1 - Install and Configure Network Policy Server

Network administrators deploy remote access servers (RAS) that give mobile users the capability to dial-up and connect to corporate resources like folders, files, and applications. For scalability, network managers add more RAS servers and deploy a modem pool to accommodate more dial-in users. However, this infrastructure upgrade will require more resources to be spent on the hardware, long distance call charges and toll-free telephone service. A company may opt to outsource their remote access services to a third party to save on telephone charges, but there is a security risk of entrusting and managing your dial-up user accounts to an external organization like Internet Service Providers (ISPs).

Fortunately, the problem of managing user accounts and security can be addressed with the use of virtual private networks and Remote Authentication Dial-in User Service (RADIUS). Virtual private networks (VPNs) makes use of an existing Internet connection to connect to corporate network resources. Therefore, a user can be located anywhere in the world and be able to gain access to network resources by using the Internet and then connecting to the VPN server. While RADIUS provides a centralized infrastructure for the authentication of dial-in VPN users, authorization for access to network resources and also counts the number of minutes a dial-in user spends in a remote session.

In this exercise, you will install the Network Policy Server (NPS) which is Microsoft’s implementation of the RADIUS protocol. You will not configure dial-up remote access server but instead, install and set up a VPN server based on Microsoft’s Routing and Remote Access Services.

To get a better understanding of this technology, please refer to your course material or use your preferred search engine to research this topic in more detail.

Task 1 - Add DNS IP Address on PLABSA01

In this task, PLABSA01 will be used as an NPS server in the PRACTICELABS.COM domain. This server has two network interfaces which are requirements for the NPS server.

To add PLABSA01 to the Practicelabs.com domain, perform the following steps:

Step 1

Ensure you have powered on the required devices and connect to PLABSA01.

On the Server Manager window, click Tools and then select Windows PowerShell.

Step 2

On the Windows PowerShell prompt, type the following command and press Enter:

netsh interface ip set dns “Ethernet” static 192.168.0.1

This command sets 192.168.0.1 as the DNS server for PLABSA01. Note that the given IP address refers to PLABDC01 - the DNS on the PracticeLabs domain. Migrating the server to this DNS server facilitates its integration into the PracticeLabs domain.

Note: If you get an error message running the above command, re-run the same command.

Task 1 - Prepare the VPN Interface

After configuring the NPS server and creating an NPS client, you need to prepare the VPN server (PLABDM02) to enable a VPN connection. In this task, you will assign the static IP address and the preferred DNS to Ethernet 2 - the second interface on PLABDM02. These settings prepare PLABDM02 as the VPN interface on the PracticeLabs domain.

To prepare the VPN server, perform the following steps:

Step 1

Ensure you have powered on the required devices and connect to PLABDM02.

From the Server Manager window, open Windows PowerShell.

Step 2

On the Windows PowerShell window, type the following command and press Enter:

netsh interface ip set address name="Ethernet 2" static 192.168.0.20 255.255.255.0
Note: The second interface will act as the VPN server’s public interface which will receive incoming connections from the VPN client.

Step 3

On the next prompt, type the following command and press Enter:

netsh interface ip set dns "Ethernet 2" static 192.168.0.1
Note: If you get an error message entering the command above, try again. This may be caused by delays in detecting the new IP address statically assigned to Ethernet 2.

This command assigns PLABDM02 Ethernet 2’s network connection with a preferred DNS of 192.168.0.1.

Task 2 - Install Remote Access Service

Task 1 - Examine the NPS Logs

NPS logs are generated when a VPN client makes a successful or failed connection to a VPN server. The created log files are in comma separated value (.csv) format. Since the authentication of the VPN client was forwarded to the NPS server (PLABSA01), the log files can be found on the mentioned server.

To examine the NPS logs, perform the following steps:

Step 1

Connect to PLABSA01.

Click the File Explorer icon on the taskbar.

Step 2

Expand This PC, then navigate to C:\Windows\System32\LogFiles.

Look for a file with a prefix of INyymmdd (the yymmdd values may vary for your session) and right-click on it, then select Open.

Note: Initially the INyymmdd file may have 0 KB size, press F5 to refresh.

Step 3

Notepad now displays an NPS log file. Notice that the log file contains entries that you will not be able to understand. For this reason, you will need an NPS log reader to be able to understand the contents of this log file.

If needed, this link explains the contents of an NPS log file:

http://technet.microsoft.com/en-us/library/cc771748(v=ws.10).aspx

Close the NPS log file.
