Exercise 1 - Extracting Hashes from a System

Attackers use various offline attack techniques to hack the target systems. Offline password attack is one of the categories in offline attacks. In this type of attack, the attacker tries to extract the password hashes from database files such as the Security Accounts Manager (SAM) in the Windows operating system. SAM is a database file in Windows operating system where the local user credentials are stored in encrypted format.

In this exercise, you will learn to extract the encrypted hashes that are present in the SAM file. You will use the pwdump tool to perform the extraction.

Learning Outcomes

After completing this exercise, you will be able to:

  • Use Pwdump Tool to Extract a Hash

Task 1 - Use Pwdump Tool to Extract a Hash

The pwdump is a hacking tool used for extracting password hashes.

In this task, you will perform the following actions to extract a hash using the pwdump tool:

  • Connecting to the PLABWIN802 virtual machine using Hyper-V Manager on PLABWIN10.
  • Booting the virtual machine using the Kali Linux ISO file.
  • Extracting the encrypted hashes from the SAM file using the pwdump tool. The tool extracts password hashes of three local user accounts such as Administrator, Guest, and Student2.
  • Creating a new text file named extracted_hashes.txt using GNU nano editor to store the extracted hashes.
  • Creating a copy of the extracted_hashes.txt file by the name tocrack.txt using GNU nano editor.
  • Retaining the hash of Student2 user account only in a tocrack.txt file and deleting the other hashes.

Step 1

Ensure you have powered on the required devices (listed in the ‘Your Devices’ section) and connect to PLABWIN10.

To start Hyper-V Manager from the PLABWIN10 desktop, on the taskbar, click the Hyper-V Manager icon.

Step 2

The Hyper-V Manager window opens.

To connect to a virtual machine, in the Virtual Machines pane, double-click PLABWIN802.

Step 3

PLABWIN802 on PLABWIN10 - Virtual Machine Connection window opens.

To select the boot file, click Media->DVD Drive->Insert Disk.

Step 4

The Open dialog box appears.

To open the Kali Linux ISO boot file, navigate to New Volume (D:)->ISO folder.

Under the ISO folder, select the kali-linux-2017.2-amd64, and click Open.

Step 5

To start the virtual machine, in the PLABWIN802 on PLABWIN10 - Virtual Machine Connection window, on the toolbar, click Start.

Step 6

Please wait while the virtual machine boots in Kali.

The KALI Boot menu is displayed.

In the boot menu, navigate using the down arrow key, and select Live (forensic mode).

Press Enter.

Note: In Live (forensic mode), the internal hard disk of the device is never used and the auto-mounting of removable media is disabled on the device.

Step 7

The Kali desktop in the virtual machine appears.

Step 8

To open the file explorer in Kali, from Kali desktop, click the Places drop-down list.

From the drop-down list, click Computer.

Step 9

The file explorer in Kali appears.

In the left pane, click Other Locations.

Step 10

To select the virtual hard drive of Windows 8 device in Hyper-V Manager, in the Other Locations pane, click 136 GB Volume.

Step 11

In 136 GB Volume, navigate to Windows->System32->config folder.

Step 12

In the config folder, observe the SAM file.

To open the SAM file in the root terminal in Kali, right-click the blank area.

A context menu appears.

From the context menu, click Open in Terminal.

Step 13

The root terminal window with the absolute path of the config folder appears.

To extract the hashes from the SAM file, at the prompt, type the following command:

pwdump SYSTEM SAM

Press Enter.

Note: In the command, the text “SYSTEM SAM” must be entered in upper case.

Step 14

The pwdump tool extracts and displays the hashes of all the local user accounts stored on the system.

For the purpose of this demonstration, the pwdump tool has extracted and displayed the hashes of three user accounts such as Administrator, Guest, and Student2.

Step 15

To copy the extracted hashes to a new text file, select the extracted hashes, and right-click.

A context menu appears.

From the context menu, click Copy.

Step 16

To navigate to the root folder, at the next prompt, type the following command:

cd\

Press Enter.

The Change Directory (cd) command enables you to change the current directory.

Step 17

To navigate to the Desktop folder, at the next prompt, type the following command:

cd Desktop/

Press Enter.

Step 18

To create a new text file on the desktop, at the next prompt, type the following command and press Enter:

nano extracted_hashes.txt

GNU nano is a text editor in Linux operating system. The command will open the text editor and create a new text file named extracted_hashes.txt.

Step 19

To paste the copied hashes in the new text file, in the GNU nano editor window, press Ctrl+Shift+v.

To write the hashes to the file, press Ctrl+o, and press Enter.

Note: If prompted to confirm the filename, press Enter.

To exit the GNU nano editor, press Ctrl+x.

Step 20

You are back at the command prompt.

To check if the new text file is created on the desktop, in the terminal and explorer windows, click the Minimize icon.

Step 21

The new text file named extracted_hashes.txt is created on the desktop.

Step 22

For the purpose of this demonstration, the password of Student2 user account from the extracted hashes will be cracked.

To crack the password, you need first to make a copy of the extracted_hashes.txt file.

Restore the terminal window from the left bar. In the terminal window, at the next prompt, type the following command:

cp extracted_hashes.txt tocrack.txt

Press Enter.

The command creates a copy of the text file extracted_hashes.txt. The new file created is tocrack.txt.

Step 23

You need to delete the hashes of Administrator and Guest accounts in the tocrack.txt file.

To edit the tocrack.txt file, at the next prompt, type the following command:

nano tocrack.txt

Press Enter.

Step 24

The tocrack.txt file opens in GNU nano editor.

To remove the Administrator and Guest account hashes, press the Delete key.

Step 25

Ensure only the hash for Student2 account is present in the file.

To write the hash to the file, press Ctrl+o, and press Enter.

Note: If prompted to confirm the filename, press Enter.

To exit the GNU nano editor, press Ctrl+x.

Step 26

You are back at the root terminal.

Exercise 2 - Cracking Extracted Hashes

To crack the extracted hashes, you will require a wordlist. A wordlist is basically a text file that contains a collection of words and is generally used in a dictionary attack.

In this exercise, you will learn to crack the hashes that were extracted in the previous exercise. You will use the default wordlist “rockyou.txt” that comes with Kali Linux. The “rockyou.txt” wordlist comes compressed by default with most popular passwords.

Learning Outcomes

After completing this exercise, you will be able to:

  • Use a Wordlist to Crack the Extracted Hash

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Windows Server 2012 R2 - Domain Controller)
  • PLABKALI01 (Kali 2016.2)
  • PLABWIN10 (Windows 10 - Domain Member)

Task 1 - Use a Wordlist to Crack the Extracted Hash

In this task, you will decompress the rockyou.txt.gz file and use the “rockyou.txt” wordlist to crack the extracted hash.

Step 1

The root terminal in PLABWIN802 on PLABWIN10 - Virtual Machine Connection window is open. Clear the window with the clear command.

To decompress the wordlist, at the prompt, type the following command and press Enter:

gzip -d /usr/share/wordlists/rockyou.txt.gz

The “rockyou.txt” wordlist is present in the default folder /usr/share/wordlists.

Exercise 3 - Cracking Passwords

There are various tools available to crack passwords. The tool “John the Ripper” is one of the password-cracking tools used to crack passwords.

In this exercise, you will learn to crack the password of Student2 user account.

Learning Outcomes

After completing this exercise, you will be able to:

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Windows Server 2012 R2 - Domain Controller)
  • PLABKALI01 (Kali 2016.2)
  • PLABWIN10 (Windows 10 - Domain Member)

Task 1 - Use a Password Cracking Tool to Crack the Password

In this task, you will use “John the Ripper” tool for cracking the password of Student2 user account. The password hash of Student2 user account is stored in the tocrack.txt file.

Step 1

The root terminal in PLABWIN802 on PLABWIN10 - Virtual Machine Connection window is open. Clear the screen with the clear command. To crack the password of Student2 user account, at the prompt, type the following command and press Enter:

john --format=NT --wordlist=/usr/share/wordlists/rockyou.txt tocrack.txt

The password-cracking tool “John the Ripper” is used in the command with the “format=NT” parameter. The “rockyou.txt” wordlist and the hash file “tocrack.txt” are passed as parameters in the command.

Step 2

The tool John the Ripper successfully cracks the password for Student2 user account, and the password is displayed.
