Exercise 1 - Install and Configure ManageEngine OpManager

ManageEngine OpManager is a network-monitoring tool. It facilitates network, server, network configuration, IP address, and other types of management. It also helps in network traffic analysis.

Learning Outcomes

After completing this exercise, you will be able to:

  • Install and Configure ManageEngine OpManager

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Windows Server 2012 R2 - Domain Controller)
  • PLABDM01 (Windows Server 2012 R2 - Member Server)
  • PLABKALI01 (Kali 2016.2)
  • PLABWIN10 (Windows 10 - Domain Member)
  • PLABWIN801 (Windows 8.1 - Domain Member)

Task 1 - Install OpManager

In this task, you will install OpManager and then use it to map the available devices in the network.

Step 1

Ensure all devices in the lab are powered on.

Connect to PLABDC01.

Close the Server Manager window.

Access the Intranet by launching Internet Explorer.

Alert: All of the devices in the lab should be powered on as they will be included in the scan later in the exercise.

Step 2

Youâ€™ll now download ManageEngine OpManager from the Intranet.

Navigate to Installation Files > Op Manager.

Click on the ManageEngine_OpManager_Free_64bit 12.0.exe link and when prompted click Save.

Step 3

Once the download is complete, to start the application setup, click Run.

Step 4

Please wait while the setup is being prepared.

Step 5

The InstallShield Wizard for ManageEngine OpManager starts.

On the Welcome page of the wizard, click Next.

Step 6

On the License Agreement page of the wizard, click Yes.

Step 7

To select the destination folder, on the Destination Folder page, keep the default settings, and click Next.

Step 8

On the Port Selection Panel page, in the WebServer box, delete the existing text, and type the following web server port:

8443

Click Next.

Step 9

On the Registration for Technical Support page, click Skip.

Step 10

ManageEngine OpManager installation starts.

Step 11

Please wait while ManageEngine OpManager configures the new software installation.

Step 12

To select the backend database for OpManager, on Select the backend database for OpManager page, ensure POSTGRESQL is selected, and click Next.

Step 13

ManageEngine OpManager message box appears.

Click OK.

Step 14

The installation of ManageEngine OpManager is completed.

To exit the wizard, click Finish.

Step 15

ManageEngine OpManager 12 Readme file opens in the Internet Explorer window.

Please wait while the initialization of OpManager is in progress.

You may need to close a File Explorer window if it opens.

After the initialization you will be directed to the link http :// plabdc01 : 8443. Alternatively, in the address bar, delete the existing text, and type the following URL:

http://plabdc01:8443

Press Enter.

Note: It may take 30 seconds or a minute to initialize. If you enter the link into the address bar before the initialization is complete, you will be shown a page that cannot be displayed.

Step 16

ManageEngine OpManager login screen is displayed.

Task 2 - Configure OpManager

After the OpManager application is installed, it is critical to configure it to serve its purpose. In this task, you will configure OpManager and then use it to map the available devices in the network.

Step 1

Ensure that the login page of OpManager is displayed. To enter the username, in the username box, type the following:

admin

In the password box, type the following password:

admin

Click Login.

Note: If prompted to save the password, click Not for this site.

Step 2

Click X to close the Get Started section.

Step 3

In the upper right of the Internet Explorer window,

Click the settings cog and hover over the Zoom setting and change it to 75%.

This is so the entire page can be seen.

Step 4

Hover over Settings in the top bar and then hover over Discovery.

Click Network Discovery.

Step 5

To scan the local network, click New Discovery.

Step 6

In the Start IP box, type the following IP address:

192.168.0.1

In the End IP box, type the following IP address:

192.168.0.255

Click Add Credentials.

Step 7

On the Add Credential pane, click Windows/WMI.

Step 8

The Add Credentials (Windows/WMI) pane appears.

In the Profile Name box, type the following credential name:

Windows Credentials

In the Domain Name\User Name box, type the following name:

PRACTICELABS.COM\Administrator

In the Password box, type the following password:

Passw0rd

In the Retype Password box, type the following text:

Passw0rd

Click Save.

Step 9

Click the dropdown for Select Credentials.

Tick the Windows box which will also tick Windows Credentials.

Type a name for the Network Discovery scan. In this example, Discovery Scan has been inputted.

Click Discover.

Step 10

The Discovery scan is now in progress.

This may take 5-10 minutes.

Step 11

The scan has completed.

A number of devices have been found. The result may be different in your lab.

Click the box next to all devices that display one. You will need to expand the entries using the arrow to see the boxes.

Click Approve.

Step 12

Hover over Dashboard in the top left and hover on Overview, click Realtime Dashboard.

This also shows the systems that were scanned.

Shutdown all virtual machines used in this lab, by using the power functions located in the Tools bar before proceeding to the next module. Alternatively, you can log out of the lab platform. 2904fde5-033a-49e0-91ee-c6e2e905aed6

Close all open windows.
