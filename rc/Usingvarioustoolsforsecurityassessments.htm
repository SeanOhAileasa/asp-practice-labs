Exercise 1 - Identifying Live Systems

The basic and foremost step in studying the physical connectivity of networks is scanning. Scanning (also known as probing) is a process of leading the target machines to reveal useful information about them. A network scanner is a tool that will aid in scanning networks. Some of the popularly used network scanner tools are nmap and hping3. Both nmap and hping3 have several features that give complete control over the scans that you perform. You can download the appropriate nmap and hping3 tools for your operating system from the following respective links:

  • http://nmap.org
  • http://www.hping.org

For this demonstration, nmap & hping3 are bundled with the default installation of Kali Linux.

In this exercise, you will learn to identify live systems using nmap and hping3.

Please refer to your course material or use your preferred search engine to research this topic in more detail.

Task 2 - Identify Live Hosts using nmap

In this task, you will use the nmap tool to identify live hosts in the Practice Labs environment. As the first step, you will run the ifconfig command in Kali Linux to display information about all network interfaces currently in operation.

To do this, perform the following steps:

Step 1

At the command prompt of the root terminal window, type the following command and press Enter:

ifconfig

As seen in the output, eth0 is the first, and the only ethernet interface in the network. lo is the loopback interface that the system uses to communicate with itself.

Find that the Kali Linux’s IP address is 192.168.0.3.

Step 2

To display the help summary page of nmap, type the following command and press Enter:

nmap -h
Note: You can view the help summary of nmap to learn various switches and scanning techniques supported by the tool.

Step 3

In nmap, you can specify the IP scan targets in the following formats:

  1. Domain name: target.com
  2. Target IP address: 192.168.0.1
  3. IP address range of the subnet: 192.168.0.1-255
  4. CIDR (Classless Inter-Domain Routing) - 192.168.0.1/24

For this demonstration, you will perform a simple ping sweep also known as an ICMP scan to determine how many hosts respond to the ping request sent by nmap on the 192.168.0.1/24 subnet.

Type the following command at the command prompt and press Enter:

nmap -sn 192.168.0.1/24

The output shows that the devices with the IP addresses 192.168. 0.1 (PLABDC01), 192.168.0.2 (PLABDM01), 192.168.0.4 (PLABWIN01), 192.168.0.5 (PLABWIN801) and 192.168.0.3 (Kali Linux) are up.

Note: The nmap switch for ping scan is ‘-sn’. The -sn switch disables port scan by default and only sends ping requests to the specified target machines. All hosts that respond to the requests with valid responses are treated as live hosts, and the rest of the hosts that do not respond to the ping request are considered as being not live.

Task 3 - Identify Live Hosts using hping3

Similar to nmap, you can also use hping3 to identify if a host is live or not. It is not possible to specify an IP address range as a target in the hping3 utility. Only one target at a time can be specified.

In this task, you will use the hping3 tool to identify live hosts in the Practice Labs environment.

To do this, perform the following steps:

Step 1

You can view the help summary page of hping3 to learn various switches and scanning techniques supported by the tool.

To display the help summary page of hping3, type the following command and press Enter:

hping3 -h

Step 2

The following command determines if the host 192.168.0.1 is live or not. To perform a simple ping scan (known as an ICMP scan) to determine if a host responds to the ping request sent by hping3, type the following command at the command prompt and press Enter:

hping3 -1 192.168.0.1
Note: The hping3 switch for ping scan is ‘-1’. This switch enables the ICMP mode in hping3 utility.

From the output, you can observe that the host 192.168.0.1 responds with the valid ICMP response for every ICMP request sent. This ensures that the host 192.168.0.1 is live.

Step 3

Press Ctrl + C to cancel the packet transmission. You can perform similar ping scans technique against other hosts in the Practice Labs environment to determine if they are live or not.

Exercise 2 - Perform Network Footprinting

In this lab, you will practice attack strategies such as footprinting.

A network scan is usually the first step in an attempt to penetrate security (or indeed to establish what needs defending). Footprinting establishes the topology and protocols deployed on the network while fingerprinting determines the services and other configuration details of a target host.

One of the most popular scanning tools is nmap. This is a command-line program operated using scripts. A GUI version (Zenmap) can perform several very useful pre-configured scans though.

To get a better understanding of this technology, please refer to your course material or use your preferred search engine to research this topic in more detail.

Task 1 - Using Nmap

To start using nmap scanner, perform the following steps:

Step 1

Ensure you are still connected to PLABKALI01.

Within the terminal window, type the following command and press Enter:

nmap 192.168.0.0/24 > hosts.txt
Note: This command will save the results in a file called hosts.txt

Step 2

When nmap is finished scanning the network 192.168.0.0/24, close the terminal window.

From the desktop of the PLABKALI device, click Show Applications icon located on the taskbar on the left.

Step 3

Scroll until you see the Leafpad icon. Click LeafPad.

Step 4

Click File then choose Open.

On the Open dialog box, under Places section, select root.

On the right details pane, select hosts.txt.

Click Open.

Step 5

Find the results of the scan of hosts in the network 192.168.0.0.

Look for a host with IP address 192.168.0.1.

Step 6

Click X to close LeafPad text editor. You should now be back on the PLABKALI01 desktop.

Double-click the Terminal icon within the toolbar.

Step 7

The terminal window opens.

To identify the operating system versions and to send scan TCP ports using the SYN packet, type the following command and press Enter:

nmap -O -sS 192.168.0.1 > hosts.txt

Press Enter.

Note: This scan will take a few minutes to complete. Also, since you are using the same file name for the output of the command, the previous file will be overwritten without any warning. Alternatively, you can use any other name for the file.

Step 8

Keep the terminal window open.

Click Show Applications menu, then scroll to select LeafPad.

Step 9

Click File then Open.

On the Open dialog box, navigate to File System>root.

On the details pane on the right, then click hosts.txt and choose Open.

Step 10

Find the results of the nmap scan on 192.168.0.1 regarding the OS details.

Close Leafpad.

Minimize the terminal window.

Task 2 - Using Zenmap

Zenmap is the GUI equivalent of nmap which is easier to use.

To start using Zenmap, perform the following steps:

Step 1

While in PLABKALI01, click Show Applications > 01 Information Gathering > and click Zenmap.

Step 2

Keep Zenmap open.

Step 3

Connect to PLABWIN10.

Right-click Wireshark and select Run as Administrator.

On the User Account Control dialog box, click Yes.

Step 4

From the top menu, click Capture and select Options.

Select Ethernet 2 and click Start.

Step 5

Switch back to PLABKALI01.

The Zenmap application is open.

Click in the Target text box, type the following IP address and click Scan:

192.168.0.1

Step 6

Zenmap runs the scan on 192.168.0.1.

Wait until the scan is completed on Zenmap (this may take a few minutes).

Step 7

When the Zenmap scan is completed, reopen Wireshark.

Step 8

Switch back to PLABWIN10.

Wait until the live capture is complete. This will take a few minutes.

From Wireshark, click Capture and select Stop.

Step 9

Click in the Filter box and type the following text and click Enter:

tcp

Alternatively, you can click the arrow next to the Filter text box.

Step 10

Scroll up to the capture file. Go to Source column and look for 192.168.0.1 (refers to a vulnerable host), and Destination 192.168.0.3 (refers to Kali Linux).

When the Zenmap port scan is run against the vulnerable host, network applications will reply with a SYN/ACK (sequence and acknowledgment) packet when a connection is made to it. The zenmap tool will send the RST (reset) to tear down the connection.

Step 11

Go back PLABKALI01.

The Zenmap application is open.

In the Target text box, type the following IP range and click Scan:

192.168.0.0/24

Click Scan.

Step 12

This will scan available hosts in the mentioned network ID 192.168.0.1/24

Note: This will take about 3 minutes to complete.

Step 13

When the scan is complete, the Hosts left column indicates the IP addresses of computers detected in the scan.

Click 192.168.0.2 to view the details about this host.

Step 14

Click on Topology tab and then click Fisheye.

The Topology tab displays the discovered network hosts detected by the scan.

Step 15

Step 16
