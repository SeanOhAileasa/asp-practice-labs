Exercise 1 - Create Windows Configuration Items for Compliance Settings

In this exercise, you will learn how to create configuration items to define configurations for managing and accessing compliance on devices.

Learning Outcomes

After completing this exercise, you will be able to:

  • Create a Configuration Item

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Domain Controller)
  • PLABSYS01 (System Center2012 Server)

Task 1 - Create a Configuration Item

In this task, you will create a configuration item to determine compliance for settings related to operating systems and its configuration.

Step 1

Ensure you have powered on the required devices and connect to PLABSYS01.

Close Server Manager.

From PLABSYS01 desktop, double-click Configuration Manager Console.

Step 2

System Center 2012 R2 Configuration Manager opens.

In the Assets and Compliance workspace, under Overview, expand Compliance Settings.

Step 3

Under Compliance Settings, select the Configuration Items node, right-click and from the context menu, select Create Configuration Item.

Step 4

Create Configuration Item Wizard opens.

On the General page, in the Name box, type:

Home Page Setting

In Assigned categories to improve searching and filtering, click Categories.

Step 5

Manage Administrative Categories dialog box opens.

In the Administrative Categories section, select IT Infrastructure.

Click OK.

Step 6

On the General page of the Create Configuration Item Wizard, click Next.

Step 7

On the Supported Platforms page, select the versions of Windows that will access this configuration item for compliance.

For this demonstration, keep the Windows Server 2012 R2 checkbox selected and deselect all other selections.

Click Next.

Step 8

On the Settings page, click New.

Step 9

Create Setting dialog box opens.

On the General page, in the Name box, type Home Page Setting.

From the Data type drop-down, select String.

Step 10

In the Specify the registry value to access compliance on computer section, click Browse.

Step 11

In the Browse Registry dialog box that opens, in the tree view pane, expand HKEY_Current_User, expand Software, expand Microsoft, expand Internet Explorer, and select Main.

In the Registry Value section, select Default Page URL.

Select the The selected registry value must not exist on client devices radio button.

Click OK.

Step 12

Back to Create Setting dialog box.

Click Apply.

Step 13

Click the Compliance Rules tab.

Select the default compliance rule and click Delete.

Step 14

With the compliance rule deleted, click New to create a new compliance rule

Step 15

In the Create Rule dialog box that opens, in the Name box, type Correct Home Page.

In the following values section, type the following text:

WWW.PRACTICELABS.COM

Select the following checkboxes:

  • Remediate noncompliance rules when supported
  • Report noncompliance if this setting instance is not found

Click OK.

Step 16

On the Create Settings dialog box:

Click Apply.

Click OK.

Step 17

On the Settings page of the Configuration Item Wizard, click Next.

Step 18

On the Compliance Rules page, click Next.

Step 19

On the Summary page, click Next.

Step 20

Wait for the process to complete.

Step 21

On the Completion page, click Close.

Step 22

Find the newly created configuration item listed in the Configuration Items section.

Exercise 2 - Create Configuration Baseline and Deploy the Baseline

You can evaluate compliance in SCCM by defining configuration baselines. Configuration baselines contain predefined configuration items that you want to evaluate. It can also contain another configuration baseline. In this exercise, you will create a configuration baseline for a Windows configuration item.

Learning Outcomes

After completing this exercise, you will be able to:

  • Create and Deploy a Configuration Baseline

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Domain Controller)
  • PLABSYS01 (System Center2012 Server)

Task 1 - Create and Deploy a Configuration Baseline

In this task, you will create a configuration baseline using the Configuration Manager on the PLABSYS01 server.

Step 1

Ensure you are connected to PLABSYS01.

In the Configuration Manager Console, ensure Assets and Compliance workspace is selected.

Under Compliance Settings, select Configuration Baselines.

On the Home tab, in the Create group, select Create Configuration Baseline.

Step 2

Create Configuration Baseline dialog box opens.

In the Name box, type

Home Page

In the Configuration Data section, click ADD and select Configuration items

Step 3

Add Configuration Items dialog box opens.

Select the configuration item that you created earlier. For example, select Home Page Setting and click Add.

Step 4

The selected configuration item gets added to the configuration baseline.

Click OK.

Step 5

Back to Create Configuration Baseline dialog box.

In the Assigned categories to improve searching and filtering section, click Categories.

Step 6

In the Manage Administrative Categories dialog box, select IT Infrastructure and click OK.

Step 7

In the Create Configuration Baseline dialog box, click OK.

Step 8

Find the newly created configuration baseline listed in the Configuration Baseline section. Deploy the configuration baseline to users through collections for evaluating its settings for compliance on a specified schedule.

Select the baseline.

On the Home tab, in the Deployment group, click Deploy.

Note: You can also deploy configuration baselines to devices through collections.

Step 9

In the Deploy Configuration Baselines dialog box that opens, in the Select the collection for this configuration baseline deployment, click Browse.

Step 10

The Select Collection dialog box opens.

Select All Users. This will be input into the Collection field.

Click OK.

Step 11

In the Deploy Configuration Baseline dialog box, keep the default schedule.

Click OK.

Step 12

The configuration baseline is deployed to All Users collection.

Exercise 3 - Create Windows Configuration Items for Compliance Settings

You can import configuration data from the Microsoft site in SCCM Configuration Packs. Microsoft and other vendors define these configuration data.

In this exercise, you will learn to import configuration data in SCCM Configuration Manager Packs from the web into Configuration Manager.

Learning Outcomes

After completing this exercise, you will be able to:

  • Import Configuration Data

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Domain Controller)
  • PLABSYS01 (System Center2012 Server)

Task 1 - Import Configuration Data

In this task, you will download configuration data from the Practice Labs Intranet Website and then import it in the Configuration Manager.

Step 1

Ensure you are connected to PLABSYS01.

Open Internet Explorer.

On the Practice Labs Intranet home page, click Installation Files.

Step 2

Click Configuration Pack from SCCM.

Step 3

Click ConfigMgr2012ConfigPack.msi.

In the notification bar, click Run.

Step 4

On the Welcome page of the installer wizard, click Next.

Step 5

On the License Agreement page, select I agree.

Click Next.

Step 6

On the Select Installation Folder page, keep the default selections and click Next.

Step 7

On the Confirm Installation page, click Next to start the installation.

Step 8

On the Installation Complete page, click Close.

Step 9

Close Internet Explorer.

Navigate to Configuration Manager Console.

In the Assets and Compliance workspace, under Compliance Settings, select Configuration Baselines if already not selected

On the Home tab, in the Create group, click Import Configuration Data.

Step 10

On the Select Files page of the Import Configuration Data Wizard, click Add.

Step 11

In the Open dialog box, browse to the following location of the configuration pack:

C:\Program Files (x86)\Configuration Pack for System Center 2012 Configuration Manager

Select CM2012ServerRolesConfigpack and click Open

Step 12

On the Select Files page of the Import Configuration Data Wizard, click Next.

Step 13

On the Summary page, click Next.

Wait for the process to complete.

Step 14

On the Confirmation page, click Close.

Step 15

Find the newly imported configuration baseline listed in the Configuration Baselines section.
