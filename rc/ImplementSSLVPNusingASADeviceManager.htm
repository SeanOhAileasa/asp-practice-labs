Exercise 1 - Implement a Clientless SSL VPN using the Cisco ASA Device Manager

In this exercise, you will use the Cisco ASDM software to create a Clientless SSL VPN enabling remote access for Web-based users to gain access to the corporate network. In this particular example, you will use PLABEXTCLI to VPN to the LDNFWASA01 firewall enabling connectivity to the PLABDMZWEB.

A base configuration has been applied to all the devices in the lab so that you can focus on configuring the VPN.

Please refer to your course material or use your favorite search engine to research for more information about this topic.

Learning Outcomes

After completing this exercise, you will be able to:

  • Configure a Clientless SSL VPN

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • LDNFWASA01 (Cisco ASA 5512)
  • PLABMGMT (Windows Server 2003 - Standalone Server)
  • PLABEXTCLI (Windows Server 2003 - Standalone Server)

Task 1 - Configure a Clientless SSL VPN

To set up a clientless SSL VPN use the following steps:

Step 1

Ensure all the devices are powered on in the lab. After PLABMGMT is up and connected to the desktop, double-click the Cisco ASDM icon on the desktop.

Step 2

Notice that the Device IP Address / Name textbox contains the following IP address:

192.168.17.1

In the Username text box, enter the following:

cisco

In the Password text box, enter the following:

cisco

Click OK.

Note: ensure Run in Demo Mode is unchecked. Remember you are connecting to a real Cisco ASA. Also, keep the Remember the username for the specified device on this computer option selected.

Step 3

The Security Warning dialog box is displayed. Select Always trust content from this publisher and then click Yes.

Step 4

The Home tab of Cisco ASDM 7.3 for ASA is displayed. Click Configuration and then on the left pane at the bottom, click Remote Access VPN

Step 5

The right pane displayed several buttons.

Note: The buttons in the right pane open help assistance. These are not the wizards for configuration.

Click Wizards, select VPN Wizards, and then select Clientless SSL VPN Wizard to start the process.

Note: On older ASDM versions, the menu item is slightly different, a screenshot of the older version is shown below for your reference.

Step 6

On the Clientless SSL VPN Connection (Step 1 of 6) page of the SSL VPN Wizard, click Next.

Step 7

On the Clientless SSL VPN Interface (Step 2 of 6) page, in Connection Profile Name text box, enter the following:

CLIENTLESS

Ensure that the SSL VPN Interface drop-down has internet selected.

Leave the other settings as default and click Next.

Step 8

On the User Authentication (Step 3 of 6) page, select Authenticate using the local user database.

The User to be added section is now enabled, in this section enter the following details:

Username: fred

Password: cisco123

Confirm: cisco123

Step 9

Click Add to add the user to the right pane. Notice that the user is now added to the right pane. Click Next.

Step 10

On the Group Policy (Step 4 of 6) page, ensure Create a new group policy is selected and enter the following text and then click Next.

CLIENTLESS_POLICY

Step 11

On the Clientless Connections Only - Bookmark List (Step 5 of 6) page, click Manage.

Step 12

In the Configure GUI Customization Objects dialog, click Add.

Step 13

In the Add Bookmark List dialog box, in the Bookmark List Name text box, enter in MYBOOKMARKS. Click Add.

Step 14

On the Select Bookmark Type dialog box, keep the default option (URL with GET or POST method) and click OK.

Step 15

In the Add Bookmark dialog box, enter in the following settings and click OK:

Title: DMZ_WEB

URL: PLABDMZWEB

Step 16

Notice that DMZ_WEB is now added to the Edit Bookmark List dialog box.

Click OK.

Step 17

Click OK at the Configure GUI Customization Objects dialog box.

Step 18

Back on the Clientless Connections Only - Bookmark List (Step 5 of 6) page, select MYBOOKMARKS from the Bookmark List drop-down list. Click Next

Step 19

On the Summary (Step 6 of 6) page, click Finish.

Step 20

The configuration will be applied to the ASA, notice using the wizard you don’t have to apply the configuration, it applies it automatically.

You are back on the Configuration tab.

Step 21

Next, connect to PLABEXTCLI.

Open Internet Explorer, and enter the following URL in the address bar:

https://213.16.48.1

Step 22

It may take a minute to load the website. However, you are prompted with a certificate error. Since you did not configure any valid certificates, this error is likely to be displayed.

Click Continue to this website (not recommended).

Step 23

The ASA login Webpage is displayed.

In the USERNAME textbox, enter the following:

fred

In the PASSWORD text box, enter the following:

cisco123

Click Login

Step 24

After the successful login, you will see the bookmark that you created previously.

Note: You will see a notification bar that prompts you to save a password. Click Not for this site.

However, notice that the DMZ_WEB URL is greyed out when you hover over the link. A popup displays the reason for the disabled DMZ_WEB. To resolve this, you will need to add a host entry in for this server as the ASA cannot resolve the address.

Step 25

Connect to the LDNFWASA01.

Use the following command to add a name at the terminal:

LDNFWASA01# configure terminal

LDNFWASA01(config)# name 172.16.16.10 PLABDMZWEB

LDNFWASA01(config)# exit

Step 26

Switch back to PLABMGMT.

Notice that the ASDM is showing an ASDM out of sync message. Whenever you configure something on the CLI and have the ASDM software open, this message will appear. You can just click Refresh Now to tell the ASDM to pull the latest configuration.

Step 27

Refresh the SSL VPN client page on PLABEXTCLI. Press F5 or right-click in the right pane and select Refresh.

Note: Some laptop keyboards require Fn + F5 keys to be pressed for refreshing a Webpage.

Step 28

Notice that the DMZ_WEB URL is no longer grey.

Click DMZ_WEB.

Exercise 2 - Implement AnyConnect using the Cisco ASA Device Manager

In this exercise, you will configure the Cisco AnyConnect client. Many users find that even though you have to download the client to the machine, it is consequently a much better experience than the Clientless SSL VPN.

Please refer to your course material or use your favorite search engine to research for more information about this topic.

Learning Outcomes

After completing this exercise, you will be able to:

  • Configure the AnyConnect Client

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABMGMT (Windows Server 2003 - Standalone Server)
  • PLABEXTCLI (Windows Server 2003 - Standalone Server)

Task 1 - Configure the AnyConnect Client

To configure the AnyConnect client, we will use a similar process as before.

Step 1

Firstly, let’s just ensure no existing identity key pairs exist.

Connect to the LDNFWASA01.

Enter the following commands to zeroize the existing identity keys at the terminal:

LDNFWASA01# configure terminal


LDNFWASA01(config)# crypto key zeroize rsa

WARNING: All RSA keys will be removed.

WARNING: All device digital certificates issued using these keys will also be removed.

Do you really want to remove these keys? [yes/no]: yes
Note: If an error appears stating Keys not found in configuration, please move onto Step 2.

Step 2

Re-login to the ASDM software on PLABMGMT, navigate to the menu and click Wizards > VPN Wizards > AnyConnect VPN Wizard:

Note: If you see a prompt to refresh, click Refresh.

Step 3

On the Introduction page of the AnyConnect VPN Connection Setup Wizard, click Next.

Step 4

On the Connection Profile Identification page, in the Connection Profile Name text box, type ANYCONNECT.

In the VPN Access Interface drop-down, ensure internet is selected.

Step 5

On the VPN Protocols page, leave the defaults selected (SSL and IPSec).

Click Manage to generate a self-signed certificate.

Step 6

Click Add in the top right of the Manage Identity Certificates window.

Step 7

Check Add a new identity certificate and click on New… next to Key Pair.

Step 8

In the Add Key Pair dialog box, click Generate Now.

Step 9

Select Generate self-signed certificate

Click on Add Certificate

Step 10

On the Enrollment Status dialog box, click OK.

Step 11

Back on the Manage Identity Certificates page, notice that a new entry has been added.

Click OK.

Step 12

On the AnyConnect VPN Connection Setup Wizard, on the VPN Protocols page, click Next.

Note: A new device certificate is now added in the Device Certificate drop-down.

Step 13

On the Client Images page, click Add.

Step 14

In the Add AnyConnect Client Image dialog box, click Browse Flash.

Step 15

In the Browse Flash dialog box, select anyconnect-win-3.1.05182-k9.pkg.

Click OK.

Step 16

On the Add AnyConnect Client Image dialog box, click OK.

Step 17

On the AnyConnect VPN Connection Setup Wizard, click Next.

Step 18

On the User Authentication page, ensure LOCAL is selected in the AAA Server Group drop-down. As we already have a user created in the previous exercise we don’t need to create any more accounts, click Next.

Step 19

On the Client Address Assignment page, click New to create a new address pool.

Step 20

In the Add IPv4 Pool dialog box, enter the following details and click OK.

Name: AnyConnect_POOL

Starting IP Address: 192.168.18.15

Ending IP Address: 192.168.18.254

Subnet Mask: 255.255.255.0

Step 21

Back on the Client Address Assignment page, click Next.

Step 22

On the Network Name Resolution Servers page, enter the following settings and then click Next.

DNS Servers: 192.168.16.10

WINS Servers: 192.168.16.10

Domain Name: practice-labs.com
Note: 192.168.16.10 is not a DNS or WINS server. However, it's not necessary for this task.

Step 23

On the NAT Exempt page, you need to exclude our inside network.

Select the Exempt VPN traffic from network address translation checkbox, ensure inside is selected in the Inside Interface drop-down.

Next to Local Network, click the ellipses button to select the correct internal address space. This selection should be any4.

Step 24

In the Browse Local Network dialog box, select the following address space:

IP Address: 192.168.16.0

Netmark: 255.255.255.0

Click OK.

Step 25

Back on the NAT Exempt page, notice that the address space has been added in the Local Network textbox.

Click Next.

Step 26

On the AnyConnect Client Deployment page, leave the defaults and allow web launch. Note that you may want to modify this in your production environment and only deploy the client manually.

Click Next.

Step 27

Finally, click Finish to deploy the configuration.

Step 28

You are back on the Configuration main page. Notice that the ANYCONNECT configuration is now added.

Step 29

In the left panel of the ASDM software, ensure Remote Access VPN is selected in the left bottom pane.

In the left pane, expand Network (Client) Access if required and then click the Group Policies node.

Step 30

Select the GroupPolicy_ANYCONNECT policy in the right pane and click Assign.

Step 31

In the Assign Group Policy dialog box, select the fred checkbox and then click OK.

Step 32

On the Group Policies page, click Apply.

Step 33

Next, connect to PLABEXTCLI.

The Clientless SSL VPN connection should have logged out by now.

Note: In case it has not logged out automatically, click Logout.

Step 34

In the address bar of the Internet Explorer, overwrite the existing URL and enter the following:

https://213.16.48.1

On the login page, in the Group drop-down, ensure ANYCONNECT is selected.

In the USERNAME textbox, enter the following:

fred

In the PASSWORD text box, enter the following:

cisco123

Click Login.

Step 35

After login, the Internet Explorer dialog box is displayed.

Click OK on the Java prompt.

Step 36

Click Install at the bottom right of the page:

Step 37

On the Internet Explorer - Security Warning dialog box, click Install.

The downloader starts to analyze the computer.

Step 38

You will receive a warning saying the untrusted VPN server has been blocked.

Click the Change Setting button.

Step 39

Click Apply Change.

You now have to click the link retry the connection visible on the page.

Step 41

The process will restart again.

This time, you will get a security warning, now you can click Connect Anyway.

Step 42

In the bottom system tray, you will notice a new icon appear (this will disappear but is still running) for the AnyConnect client. It looks like a small globe.

Step 43

Click the Connect Anyway button again. An error will appear. Click OK.

Step 44

In the lab, you are using a remote desktop connection to connect to the ASA. Therefore, you need to allow this in policy.

Connect to PLABMGMT.

In the ASDM navigate to Configuration > Remote Access VPN > Network (Client) Access > AnyConnect Client Profile.

Note: If an error related to admin access appears, click OK.

Click the Edit button.

Step 45

In the AnyConnect Client Profile Editor dialog box, scroll down the options until you find Windows VPN Establishment. Change this drop-down from Local users only to AllowRemoteUsers.

Step 46

On the AnyConnect Client Profile Editor dialog box, click OK.

Step 47

Flip back to PLABEXTCLI.

Click OK to any errors or warnings that are on the page.

Step 48

In the bottom right should be the Cisco AnyConnect client. If it’s not there, start it from Start > All Programs > Cisco > Cisco AnyConnect Secure Mobility Client.

Step 49

In the Cisco AnyConnect Secure Mobility Client dialog box, notice the VPN server name mentioned is LDNFWASA01 (IPsec) IPv4.

Click Connect.

You will see the security warning again.

Click Connect Anyway.

Step 51

Enter the following credentials and click OK:

Username:

fred

Password:

cisco123

Step 52

You may get the security warning again. If you do, click Connect Anyway.

Step 53

You will notice that the connection freezes!

This is because the routing is modified by the VPN client and you can no longer connect. After a few minutes, the RDP connection will drop (try clicking in the window or moving the mouse around).

Press OK to this error message.

Step 54

Reconnect to PLABEXTCLI by clicking the On button.

Note: You are likely to see an error message relating to the disconnection of remote console. Click OK to close the error message

Step 55

To configure split tunneling, connect to PLABMGMT and in the ASDM (in the left pane), navigate to Group Policies under Network (Client) Access.

Select the GroupPolicy_ANYCONNECT profile and click Edit.

Step 56

In the left pane, expand Advanced and select Split Tunneling.

Step 57

Deselect Policy: inherit and ensure Policy mentions Exclude Network List Below.

Deselect Network List: inherit then scroll to the right and click the Manage button next to the Network List box.

Note: You will need to scroll to the right to see the Manage button.

Step 58

In the ACL Manager dialog box, click Add and select Add ACL.

Step 59

In the Add ACL dialog box, type split_tunnel in the ACL Name text box.

Click OK.

Step 60

Notice that split tunneling is now added in the ACL Manager dialog box. Select the ACL and then click the Add down arrow again; this time, select Add ACE.

Step 61

On the Add ACE dialog box, enter in 10.44.24.0/24 in the Address drop-down. Ensure the Permit action is selected.

Click OK.

Step 62

On the ACL Manager dialog box, click OK.

Step 63

Click OK to any open dialog boxes.

On the Configuration page, click Apply when back at the main ASDM page.

Step 64

Switch back to PLABEXTCLI and reconnect the VPN again.

Again, you will be prompted with the security warning and your credentials. Click Connect anyway and enter the following credentials:

In the USERNAME textbox, enter the following:

fred

In the PASSWORD text box, enter the following:

cisco123

Click Login.

Notice that this time the connection succeeds.

Note: In the real world, if you have a valid security certificate, this installation is a little smoother. However, its good practice to have seen a few issues in this lab to get the client up and running!

Step 65

To fully test you are connected, you need to now test remote desktop from PLABEXTCLI to PLABMGMT at 192.168.16.10.

Click Start and select Run.

Type the following command in the Open drop-down and click OK:

mstsc

Step 66

In the Remote Desktop Connection dialog box, enter the following IP address in the Computer drop-down and click Connect:

192.168.16.10

Step 67

In the Windows Security dialog box, enter the following credentials:

Administrator
Passw0rd

Click OK.

Step 68

In the Remote Desktop Connection dialog box, a certificate warning is displayed. Click Yes.

You are now connected to 192.168.16.10. This proves that the split tunnel is working and VPN is operational.
