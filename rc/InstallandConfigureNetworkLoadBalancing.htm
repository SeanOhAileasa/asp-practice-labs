Exercise 1 - Installing Network Load Balancing Nodes

Network Load Balancing (NLB) service is a feature included in Windows Server 2016 and earlier versions. This service balances network traffic load sent to the NLB cluster on a TCP/IP network. The NLB cluster is a configuration of multiple virtual IP addresses. The network traffic load is balanced among the multiple servers on the cluster. Moreover, load balancers provide built-in redundancy by sensing any device failures on the network and redistributing their load over other devices. Load balancers are normally used to ensure reliable delivery of mission-critical web applications to a large user population base.

Please refer to your course material or use your favorite search engine to research for more information about this topic.

Exercise 4 - Examining the Working of an NLB Cluster

NLB manages traffic to each host on the cluster by distributing the client requests as per the rules configured for the cluster. The client requests can be from a single client or from multiple clients. The hosts on the cluster receive the requests and respond according to the configuration rules specified for each of them.

To get a better understanding of this technology, please refer to your course material or use your preferred search engine to research this topic in more detail.

Exercise 5 - Configuring and Validating Port Rules for NLB

Port rules for an NLB control the traffic management and working of the cluster. Any host added to the cluster after the first host automatically inherits the cluster port rules from the initial host.

Rules specify the values for parameters including the IP addresses, the port range, and the protocol of operation.

Task 1 - Configure Port Rules

The NLB port rules control how the NLB cluster functions. A port rule mainly defines the following two operational parameters:

  • Affinity - This is a method used to associate client requests to cluster hosts. When no affinity is specified, all network requests by clients are load-balanced across the cluster without regard to their source.
  • Filtering mode - Can be a single host or multiple hosts. This specifies how the cluster handles traffic described by the port range and protocols.

In this task, you will define the port rules for the two hosts added to the cluster. To configure port rules, perform the following steps:

Step 1

Connect to PLABDM01 and open the previously minimized PowerShell window. If PowerShell window was closed, open it once again. Clear the screen with the clear command.

Type the following command and press Enter:

md nlbport

Notice that this creates the nlbport folder under the C:

Step 2

Clear the screen with the clear command. You will now copy the contents of \inetpub\wwwroot to the nlbport folder.

On the next command prompt, type the following command:

xcopy /s c:\inetpub\wwwroot c:\nlbport

Press Enter.

Step 3

Notice the command copied the following two files to the nlbport folder:

iisstart.png

iisstart.htm

New-Website -Name nlbport -PhysicalPath "c:\nlbport" -port 6789

Task 2 - Validate Port Rules

Port rules by default, distribute the traffic load equally to all the servers on the cluster, until otherwise specified. In this task, you will access the plabdm01-nlb cluster on the PLABDM02 workstation and verify that the image is fed to the nlb alternately by both the hosts on the cluster.

To validate the port rules, perform the following steps:

Step 1

Connect to PLABDM02, close Server Manager, and click Internet Explorer.

Step 2

In the address bar on the browser, type the following in the address bar and press Enter:

http://plabdm01-nlb
Note: You may be prompted with the Internet Explorer 11 dialog box. Select Use recommended security and compatibility settings and click OK.

Step 3

Notice that the web page displayed is the one without the text NLB-Test.

Note: The web page that you may get displayed on your lab can be the one with the “NLB-Test” keyword. NLB uses a distributed algorithm to map incoming clients to the member servers of the cluster based on their IP address and port. When an NLB cluster receives an incoming request, all hosts in the NLB cluster perform a mapping to determine which host should handle the request quickly.

Step 4

Press F5 to refresh the Web page.

Notice that the other Web page, with the NLB Test text, appears.

This confirms that Web content is delivered alternately by PLABDM01 and PLABSA01 through the NLB service.

Important: Clicking the Refresh button again will not immediately switch you back to the web page that was displayed initially, because NLB acts as a filter between the cluster adapter driver and the TCP/IP protocol stack in order to distribute the traffic between hosts in the NLB cluster. There will be a lag if you attempt to refresh. However, you can try a few mouse clicks on the Refresh button to manually refresh the web page. Keep going until the web page changes.

Close Internet Explorer.

Task 3 - Validate High Availability of NLB Cluster

NLB cluster ensures high availability of resources to various applications running on the cluster. NLB hosts emit and receive “alive” signals from other hosts on the cluster. If hosts do not receive the “alive” signal from any of the host(s) on the cluster, the surviving hosts converge on the cluster to take on the load of the failed host. In this task, you will remove one of the servers from the cluster and analyze the effect of the same on the output.

To validate the high availability of the NLB cluster, perform the following steps:

Step 1

On your computer, access the Practice Labs Web application.

Select the PLABDM01 device and then click the || Suspend button.

Wait until PLABDM01 is fully suspended. You should get the Power on option.

Note: In the Practice Labs environment, when you suspend a running device, you are left with only two choices: either Power On or Refresh. Power On will bring the device back from the Suspended state. Refresh will refresh the device but it does not bring it out of the Suspended state.

Step 2

Switch to PLABDM02 and reopen Internet Explorer if required.

In the address bar on the browser, type http://plabdm01-nlb

Press Enter. Press F5 a few times to refresh the display.

Notice that the text NLB-Test does not appear on the image because the PLABDM01 device is currently switched off. Therefore, all web content comes only from PLABSA01
