Exercise 1 - Working with RDP

Remote Desktop is a built-in remote management tool in Windows 10. This tool allows you to connect to another computer’s desktop using the Remote Desktop Protocol (RDP).

When connected to another computer’s desktop via RDP, you can further investigate problems reported by the users and make the necessary remediation to fix any errors. By default, Administrators can RDP a Windows device and take over the system control using the tool. This means that any user currently signed on to Windows will be disconnected from their current session, should the administrator proceed to sign on via RDP. By default, the Remote Desktop feature in Windows 10 is enabled.

In this exercise, you will work with RDP and connect computers remotely.

Please refer to your course material or use your preferred search engine to research this topic in more detail.

Exercise 2 - Administering Windows with PowerShell Remoting

Windows PowerShell can remotely manage a Windows computer by enabling a feature called PowerShell Remoting. This feature can be useful when using a GUI-based tool that is not viable due to the low network bandwidth situation between two locations.

A service called Windows Remote Management (WinRM) makes a Windows device capable of being managed remotely. You need to set the appropriate firewall rules on both sides of the network or hosts to make PowerShell remoting possible. When connected via PowerShell remoting, you can run cmdlets and scripts as if you are running those commands interactively on the local computer. However, the Windows Firewall exception for the WinRM listener will fail if any of the network cards are configured to have a public type. The firewall will refuse to create exceptions on public network cards. Hence, you must change the network card type to either private or domain. Alternatively, you can disable the public network connections for establishing PowerShell Remoting.

In this exercise, you will configure and enable Windows PowerShell remoting between PLABWIN10 and PLABWIN801. You will remotely access PLABWIN801 from PLABWIN10 and transmit commands to PLABWIN801. As a pre-requisite to this task, you will first disable public network connections in PLABWIN10 and PLABWIN801 machines for establishing PowerShell Remoting.

To learn more about Windows PowerShell remoting, please refer to your course material or use your preferred search engine to research this topic in more detail.

Task 3 - Use Windows PowerShell to Manage Remote Computers

Windows PowerShell remoting is disabled by default on Windows 10 operating system. You need to enable it to perform remoting manually. You can run the WinRm quickconfig command to enable Windows Remote Management service. The enable-PSremoting -force command enables Windows PowerShell remoting.

In this task, you will learn to use the various cmdlets to manage remote computers.

Step 1

Switch to PLABWIN10.

Click on the Windows Search.

Type the following text:

Services

Step 2

From the Best match pop-up menu, click Services.

Step 3

Scroll down the list to find Windows Remote Management (WS-Management). Right-click and select Properties.

Step 4

Set Startup Type to Automatic.

Click on the Start button.

Click Apply and OK.

Step 5

Notice that the startup type is now set to Automatic, and service is now running.

Step 6

Close the Services window.

Step 7

In the search, type the following text:

Windows PowerShell

Step 8

From the Best match popup, right-click Windows PowerShell and select Run as administrator.

Step 9

Note: In the User Account Control dialog box, click Yes.

It may take a moment for the device to load up PowerShell. When it does, ensure Administrator Windows PowerShell is open.

At the command prompt, type the following command to enable WinRm and press Enter:

winrm quickconfig

Step 10

You will be prompted to enable the Windows Firewall exception for the winrm listener.

Type:

Y

Then, press Enter.

Windows PowerShell has updated WinRM for remote management and has enabled WinRM firewall exception.

Step 11

To confirm WinRm is enabled, type the following command to enable PowerShell remoting and press Enter:

enable-psremoting -force

You will see the following output:

WinRM is already set up to receive requests on this computer.

WinRM is already set up for remote management on this computer.

Step 12

To configure the trusted hosts, type the following command and press Enter:

set-item wsman:\localhost\client\trustedhosts *
Note: The * parameter allows PLABWIN10 to connect with any computer.

Step 13

At the prompt, type:

Y

Then, press Enter.

Step 14

Step 15

The Search pane opens.

In the search box, type:

powershell

Step 16

Note: When the User Account Control dialogue box appears, click Yes.

Administrator: Windows PowerShell displays.

winrm quickconfig

Y  

Step 17
