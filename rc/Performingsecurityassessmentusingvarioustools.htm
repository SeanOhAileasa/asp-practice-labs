Exercise 1 - Identifying Live Systems

The basic and foremost step in studying the physical connectivity of networks is scanning. Scanning, also known as probing, is a process of leading the target machines to reveal useful information about them. A network scanner is a tool that will aid in scanning networks. Some of the popularly used network scanner tools are nmap and hping3. Both nmap and hping3 have several features that give complete control over the scans that you perform. You can download the appropriate nmap and hping3 tools for your operating system from the following respective links:

  • http://nmap.org
  • http://www.hping.org

For this demonstration, nmap & hping3 are bundled with the default installation of Kali Linux.

In this exercise, you will learn to identify live systems using nmap and hping3.

Please refer to your course material or use your preferred search engine to research this topic in more detail.

Task 2 - Identify Live Hosts using nmap

In this task, you will use the nmap tool to identify live hosts in the Practice Labs environment. As the first step, you will run the ifconfig command in Kali Linux to display information about all network interfaces currently in operation.

Step 1

Click the Kali icon located at the top left.

Point to Usual applications > System Tools > QTerminal

Click the result of QTerminal.

Step 2

At the command prompt of the root terminal window, type the following command:

ifconfig

Press Enter.

As seen in the output, eth0 is the first, and the only ethernet interface in the network and lo is the loopback interface that the system uses to communicate with itself.

Find that the Kali Linux’s IP address is 192.168.0.3.

Step 3

To display the help summary page of nmap, type the following command:

nmap -h

Press Enter.

Note: You can view the help summary of nmap to learn various switches and scanning techniques supported by the tool.

Step 4

In nmap, you can specify the IP scan targets in the following formats:

  1. Domain name: target.com
  2. Target IP address: 192.168.0.1
  3. IP address range of the subnet: 192.168.0.1-255
  4. CIDR (Classless Inter-Domain Routing) - 192.168.0.1/24

For this demonstration, you will perform a simple ping sweep, also known as the ICMP scan, to determine how many hosts respond to the ping request sent by nmap on the 192.168.0.1/24 subnet.

Type the following command at the command prompt:

nmap -sn 192.168.0.1/24

Press Enter.

The output shows that the devices with the IP address 192.168. 0.1 (PLABDC01), 192.168.0.2 (PLABDM01), 192.168.0.4 (PLABWIN01), 192.168.0.5 (PLABWIN801) and 192.168.0.3 (Kali Linux) are up.

Note: The nmap switch for the ping scan is ‘-sn.’ The -sn switch disables port scan by default and only sends ping requests to the specified target machines. All hosts that respond to the requests with valid responses are treated as live hosts, and the rest of the hosts that do not respond to the ping request are considered as being not live. Task 3 - Identify Live Hosts using hping3

Similar to nmap, you can also use hping3 to identify if a host is live or not. It is not possible to specify an IP address range as a target in the hping3 utility. Only one target at a time can be specified.

In this task, you will use the hping3 tool to identify live hosts in the Practice Labs environment.

Step 1

Clear the screen using the clear command.

You can view the help summary page of hping3 to learn various switches and scanning techniques supported by the tool.

To display the help summary page of hping3, type the following command:

hping3 -h

Press Enter.

Step 2

To perform a simple ping scan (known as an ICMP scan) to determine if a host responds to the ping request sent by hping3, type the following command at the command prompt:

hping3 -1 192.168.0.1

This command determines if the host 192.168.0.1 is live or not.

Press Enter.

Note: The hping3 switch for the ping scan is ‘-1’. This switch enables the ICMP mode in hping3 utility.

From the output, you can observe that the host 192.168.0.1 responds with a valid ICMP response for every ICMP request sent. This ensures that the host 192.168.0.1 is live.

Step 3

Press Ctrl+C to cancel the packet transmission.

You can perform similar ping scans technique against other hosts in the Practice Labs environment to determine if they are live or not.

Exercise 2 - Performing Zone Transfers

The DNS (Domain Name System) server service provides three types of zones, such as primary, secondary, and stub zones. Zone transfer is a mechanism in which the changes that are made to the zone on a master server are replicated on all the secondary servers for that particular zone.

In this exercise, you will learn to perform zone transfers using command-line tools such as nslookup and dig.

Please refer to your course material or use your preferred search engine to research this topic in more detail.

Exercise 6 - Use the Social Engineering Toolkit (SET) in Kali Linux

With a little skillset, you can always break someone’s password using various methods, such as brute-force or dictionary attack. However, it is much easier to trick someone to simply share the password with you without them even knowing that they have shared it. You can also control their system without letting them know, but you should trick them into installing something or opening a file, which contains a specific payload. When you perform such tricks with people, to share their confidential information, or run a malicious file or payload, you are performing a social engineering attack, which is an art of manipulating people to perform a task that allows you to gain their confidential information or even control their system.

Task 1 - Using the Netstat command-line tool

The routing table is a key piece of information to display topologies directly related to the device and its surrounding environment. It details types of adapters used to make connections, the gateways that traffic is flowing through, and primarily it is used as a troubleshooting tool when investigating where traffic bottlenecks might be occurring.

In this task, you will be using the Netstat command-line tool to display the routing tables.

Step 1

Ensure you have powered on all the devices listed in the introduction and connect to PLABWIN10.

Step 2

Right-click Start and select Command Prompt.

Step 3

To display the routing table, type the following command and press Enter:

netstat -r

The output of the netstat -r command is displayed. The output of this command is divided into several sections, such as:

  • Interface list
  • IPv4 Route Table
  • Persistent Routes
  • IPv6 Routing Table
  • Persistent Routes

Task 2 - Using the Route Tool

In this task, you will be using the route command to display the route information, similar to the netstat -r command. It is useful to know a variety of commands which have similar functions.

To use the route tool, perform the following steps:

Step 1

Restore the command prompt window from the taskbar.

Type the following command and press Enter:

cls

Step 2

Type the following command and press Enter:

route PRINT

The output of the route print command is displayed. The output of this command is divided into several sections - just like netstat -r. The output contains the following sections:

  • Interface list
  • IPv4 Route Table
  • Persistent Routes
  • IPv6 Routing Table
  • Persistent Routes

You are presented with an Interface List attached to the device, which can be key to learning more about the network configurations.

Importantly, you are then presented with an IPv4 Route Table, which shows the Active Routes being applied to the network adapters. Finally, the output shows if there are any persistent routes on the devices. The route command displays if no persistent routing paths are being used with this device on IPv6.
