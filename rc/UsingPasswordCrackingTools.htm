Task 2 - Install Cain & Abel

Cain & Abel is a password cracking utility that helps to recover passwords from various sources, such as a local system or wireless.

Alert: You must perform this installation on the virtual machine named PLABWIN701. Cain and Abel will not function correctly on PLABDM01.

To install Cain & Abel, perform the following steps:

Step 1

Ensure that the required devices in the lab environment are powered on. Connect to PLABWIN701 and launch Internet Explorer.

Step 2

From the Tools and resources intranet Webpage, click Tools.

Note: The size of each folder may differ when you access this Webpage. This is because the tools are added or removed from their respective folders.

Step 3

From [..] > Tools page, scroll down and click Hacking Tools.

Step 4

On the [..] > Tools > Hacking Tools page, scroll down and click on the WinPcap_4_1_3.exe link.

Click Save on the information bar.

Step 5

Once the download has been completed, close the information bar at the bottom of the page.

Step 6

Next, scroll up and click on the Cain_original.zip link.

From the information bar, click Save.

Step 7

Once the download is complete, click Open folder.

Step 8

On the Downloads window, double-click on WinPcap_4_1_3 application.

Note: If presented with a User Account Control dialog box regarding making changes to the computer, click Yes.

Step 9

On the WinPcap 4.1.3 Setup dialog box, click Next.

Step 10

On the License Agreement page, click I Agree.

Step 11

On the Installation options page, click Install.

Step 12

Once the Installation is complete, click Finish.

Step 13

Back on the Downloads window, right-click on the Cain_original zipped folder and select Extract All.

Step 14

Accept the default path and click Extract.

Step 15

On the Cain_original window, double-click the Cain_original application.

Alert: You MUST run Cain and Abel on PLABWIN701. This step will not work if it is carried out on PLABDM01. Note: If presented with a User Account Control dialog box regarding making changes to the computer, click Yes.

Step 16

The Cain application window opens.

Keep the Cain application window open.

Close the Internet Explorer window.

Task 3 - Using Cain & Abel

In this task, you will start exploring the features of Cain & Abel password cracking tool.

To start using Cain & Abel, do the following steps:

Step 1

The Cain window is displayed. Click the Cracker tab.

Note: It is better to maximize the Cain window and work. You will be able to view all options without scrolling.

Step 2

From the left pane, select LM & NTLM Hashes Click the Add to list (+) sign on the top bar.

Step 3

On the Add NT Hashes from wizard, select Import Hashes from the local system and click Next.

Step 4

Note that a number of user accounts are displayed.

Step 5

In the right pane, right-click Administrator, select Brute-Force Attack, and then select NTLM Hashes.

Step 6

The Brute-Force Attack dialog box is displayed.

The Predefined textbox contains a string of characters.

Step 7

In the Password length drop-downs, set Max to 8 and click Start.

Step 8

The password cracking process is initiated. Note the time in the Time Left field. This time depends on the value defined in the Max drop-down and the charset that you choose. The higher the value, longer the time to crack a password. Also, more combinations you provide in the charset, more time it will take to use the combinations and crack the password.

Keep the Cain & Abel application running.

Task 4 - Using PWdump7

PWdum7 is a command-based utility that is used to recover password hashes from a local system as well as a remote system. You can dump the hashes in a .txt file and then use tools, such as Cain & Abel or John the Ripper, to crack the hashes.

To use PWdump7, perform the following steps:

Step 1

From PLABWIN701, go to system tray and expand it.

Then, right-click AVG and select Open AVG User Interface.

Step 2

Click Tools and select Advanced Settings…

Step 3

On the Advanced AVG Settings dialog box, from the left navigation pane, click Temporarily disable AVG protection.

On the right details pane, select Temporarily disable AVG protection checkbox.

Step 4

Click OK.

Step 5

On the Temporarily disable AVG protection page of AVG Anti-Virus Edition 2012, keep the default timing of 15 minutes and click OK.

Note: If you are prompted with the User Account Control dialog box, click Yes.

Step 6

Once again, open Advanced settings from Tools menu. On the left navigation pane, expand Anti-Virus if required and select Resident Shield.

On the right details pane, deselect the Enable Resident Shield checkbox.

Click OK.

Note: If it is already disabled, move on to the next step.

Step 7

Minimize AVG Anti-Virus window.

Step 8

Click Start and in Search programs and files box, type cmd. Right-click cmd and select Run as administrator.

Note: If you are prompted with the User Account Control dialog box, click Yes.

Step 9

In the command prompt window, type:

r:

Press Enter.

Step 10

To change directory to pwdump7, type:

cd pwdump7

Press Enter.

Step 11

Now type:

pwdump7

Note that the hashes are now displayed.

Step 12

Let's export the hashes into a .txt file.

Enter the following command:

pwdump7 > c:\password.txt

Step 13

Let's confirm if the password.txt file is now generated.

Type the following command and press Enter.

notepad c:\password.txt

Step 14

Notepad shows the content of the exported passwords.

Exit from Notepad after viewing the contents of the text file.

Minimize the command prompt window.  

Step 15

Go back to Cain & Abel.

If the Brute-Force Attack is still on-going, click Stop.

Step 16

Click Exit to close Brute-Force attack dialog box

Step 17

Ensure that you are on the Cracker tab.

Step 19

Step 20

Step 21

Step 22

The Add NT Hashes from the wizard is displayed.

Step 23

Step 24

Task 5 - Disable LM HASH

LM hashes were stored in the SAM database with the older operating systems, such as Windows XP. However, from Windows Vista onwards, LM hashes were no longer stored. In the registry, the registry key, NoLMHash, prevented the storage of LM hashes in the SAM database.

If you have systems with Windows XP on your network, you can force disabling LM hashes storage in SAM database through the Group Policy.

To disable, perform the following steps:

Step 1

Open the PLABDC01 device. The Server Manager window opens by default.

Step 2

In Server Manager, click Tools, then select Group Policy Management.

Step 3

The Group Policy Management window is displayed. In the left pane, expand Forest: PRACTICELABS.COM and expand Domains. Further, expand PRACTICELABS.COM.

From the left pane, right-click Default Domain Policy and select Edit.

Step 4

The Group Policy Management Editor window is displayed.

From the left pane, expand Policies under Computer Configuration, expand Windows Settings, expand Security Settings, expand Local Policies, and then select Security Options.

Step 5

Go to the right details pane and scroll down the list.

Look for the following policy setting.

Note that Group Policy has the following policy enabled: Network security: Do not store LAN Manager hash value on next password change

Step 6

You need to set the Group Policy to ensure network authentication is using a secure method.

Locate and double-click the following policy: Network security: LAN Manager authentication level.

Step 7

The Network security: LAN Manager authentication level Properties dialog box is displayed.

Step 8

Select Define this policy setting and then select Send NTLMv2 response only. Refuse LM & NTLM from the drop-down.

Click OK.

Step 9

The Confirm Setting Change dialog box is displayed.

Click Yes.

Step 10

The policy is now enabled.

Close Group Policy Management Editor window.

Step 11

Close Group Policy Management snap-in window.
