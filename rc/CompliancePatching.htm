Exercise 1 - Install and Configure WSUS

Windows Server Update Services (WSUS) provides a cost-effective patch management solution to deploy updates to domain-joined Windows servers and workstation in a corporate network. WSUS is fully integrated in Windows Server 2012 and can be enabled on Windows clients by configuring settings in Group Policy Objects - GPO.

Exercise 2 - WSUS Server Certificates Security

Security for certificates is a critical component to making sure information is kept confidential. By doing this, we can understand who has delivered said information, thereby confirming the integrity of not only the data, but also the sender. By applying the secure sockets layer, we help to ensure that the server is using the required security channel to communicate on.

Learning Outcomes

After completing this exercise, you will be able to:

  • Enable SSL for WSUS Server
  • Exporting Self-Signed Certificates
  • Configuring Domain to Trust Self-Signed Certificates

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Windows Server 2012 R2 - Domain Controller)
  • PLABWIN801 (Windows 8.1 - Domain Member)

Task 1 - Enable SSL for WSUS Server

For security reasons, it is important that you enable secure sockets layer for the WSUS server to encrypt web network traffic between the server and the requesting clients. In addition, SSL verifies the identity of the WSUS server when Windows update clients make a connection to it.

To enable SSL on the WSUS server, perform the following steps:

Step 1

On the PLABDC01 device, go to the Server Manager Dashboard, click on Tools and select Internet Information Services Manager.

On IIS Manager, click PLABDC01.

Note: If you get a prompt about updating to latest web platform components, click No.

From the middle pane, select Server Certificates.

On the Actions pane, click Open Feature.

Step 2

On the Actions pane, click Create Self-Signed Certificate.

Step 3

On the Specify Friendly Name page of the Create Self-Signed Certificate dialog box, type the following name in the Specify a friendly name for the certificate text box:

PLABDC01-WSUS

In the Select a certificate store for the new certificate drop-down, ensure Personal is selected.

Click OK.

Step 4

Expand Sites in the left pane and then click WSUS Administration.

On the center pane, right-click SSL Settings and select Bindings.

Step 5

On the Site Bindings dialog box, select https and click the Edit button.

Step 6

On the Edit Site Binding dialog box, use the following information:

Hostname: 

plabdc01.practicelabs.com

SSL certificate: 

PLABDC01-WSUS

Click OK.

Step 7

Click Close to save changes in Site Bindings.

Step 8

Back in the IIS Manager, ensure WSUS Administration is selected in the left pane. In the middle pane, make sure that SSL Settings is selected and click Open Feature in the right pane.

Step 9

On the SSL Settings page, select the Require SSL checkbox.

Then, on the Actions pane, click Apply.

The required changes are now saved.

Exercise 3 - Create Computer Groups for WSUS

Computer groups enable you to target updates to specific computers and can allow for specific updates to be applied to specific computers in a granular fashion. This significantly speeds up computer administration processes and improves security by allowing for mass deployment of security patches to all affected machines in one movement. In this exercise, you will go through the steps of creating and assigning these groups.

Exercise 4 - Configure GPO Policy for WSUS

In the following exercise, you will implement the tasks to deploy WSUS updates via Group Policy. Using Group Policy, we can designate the services and permissions for an update to affect computer groups of specific departments for example. However, changing relationships in Group Policy can have significant effects, and so care must be taken when working with this method of configuring devices.

Learning Outcomes

After completing this exercise, you will be able to:

  • Create a GPO for Update Services Client
  • Troubleshoot Password and Verify WSUS Client Functionality

Your Devices

You will be using the following devices in this lab. Please power these on now.

  • PLABDC01 (Windows Server 2012 R2 - Domain Controller)
  • PLABWIN801 (Windows 8.1 - Domain Member)

Task 1 - Create a GPO for Update Services Client

To configure a GPO (Group Policy Object) for WSUS client, perform the following steps:

Step 1

Connect to PLABDC01 device.

In Server Manager, click Tools and select Active Directory Users and Computers.

Step 2

Right-click PRACTICELABS.COM, select New and then select Organizational Unit.

Step 3

On the New Object-Organizational Unit dialog box, type WSUS Clients and click OK.

Step 4

Navigate to the Computers container in the left pane, select PLABWIN801, then right-click and select Move.

Step 5

On the Move dialog box, select WSUS Clients and click OK.

Note: PLABWIN801 will be used as test computer for distribution of updates via WSUS.

Step 6

Select WSUS Clients in the left pane.

Verify that the one computer (PLABWIN801) is present.

Close Active Directory Users and Computers.

Step 7

From Server Manager, click Tools and select Group Policy Management.

Step 8

Expand PRACTICELABS.COM.

Right-click WSUS Clients, then select Create a GPO in this domain, and Link it here.

Step 9

On the New GPO dialog box, type WSUS Updates and click OK.

Step 10

Right-click the WSUS Updates GPO link and select Edit.

Step 11

On the Group Policy Management Editor window, navigate in the left pane to Computer Configuration > Policies > Administrative Templates > Windows Components and select Windows Update.

https://plabdc01.practicelabs.com:8531

To verify if the WSUS GPO policies apply to the client, follow these steps:
